# Brand System Design

Create comprehensive, AI-ready brand system documents that serve as the single source of truth for visual identity, brand voice, content strategy, and creative direction.

## When To Use

- Building a new brand from scratch
- Rebranding an existing business
- Creating a creative direction brief for designers or agencies
- Documenting brand standards for social media managers, ad buyers, or AI content generators
- Onboarding new team members who need to produce on-brand content
- Preparing for investor pitches or partnership discussions

## Output Format

Default deliverable: a single comprehensive markdown file named `{brand-name}-brand-system.md`.

For multi-brand portfolios or agencies: one file per brand, stored in a dedicated repo or directory.

## Document Structure

The brand system document should include these sections. Adapt depth based on brand maturity and use case.

### 1. Brand Overview

| Element | What to capture |
|---------|-----------------|
| Brand name | Exact spelling, capitalization, abbreviations |
| Tagline | Primary and secondary taglines |
| Mission | What the brand exists to do |
| Positioning | Where it sits in the market |
| Core philosophy | The belief that drives everything |
| Unique selling proposition | The one thing no competitor can claim |
| Target perception | How the audience should feel when they encounter the brand |

**Quality check:** Can a stranger read this section and explain what the brand does in one sentence?

### 2. Target Audience

| Element | What to capture |
|---------|-----------------|
| Ideal customer profile | Role, company stage, revenue range, decision-making authority |
| Industries | Specific sectors, not "B2B" |
| Pain points | The problems that keep them awake |
| Desires | What they want, stated in their words |
| Buying psychology | What triggers a purchase decision |
| Emotional triggers | The feelings that drive action |
| Common frustrations | What they've tried and hated |

**Quality check:** Would the ICP read this and say "yes, that's me"?

### 3. Brand Voice

| Element | What to define |
|---------|----------------|
| Tone | Confident, playful, authoritative, irreverent, etc. |
| Writing style | Sentence length, active vs. passive, specificity rules |
| Messaging style | How claims are structured (outcome-first, quantified, etc.) |
| CTA style | Direct, curiosity-driven, exclusive, risk-reversal |
| Sales tone | Consultative, decisive, challenger, etc. |
| Words frequently used | The vocabulary that sounds like the brand |
| Words to avoid | Generic corporate language, competitor terms, clichés |

Include multiple examples for each element. Good examples and bad examples side by side are more useful than abstract rules.

### 4. Visual Identity System

| Element | What to specify |
|---------|-----------------|
| Primary colors | 2–3 colors with HEX codes and usage rules |
| Secondary colors | Supporting neutrals and surfaces |
| Accent colors | 2–4 accent colors for data, success, warnings, premium |
| Typography system | Font families, weights, sizes, line heights, letter spacing |
| Font hierarchy | Display, H1–H4, body, caption, mono, stat — with exact values |
| UI style | Dark/light default, card treatment, border style |
| Spacing system | Token values (xs, sm, md, lg, xl, etc.) |
| Border radius | Token values for buttons, cards, images, pills |
| Shadow style | Prefer glow effects over drop shadows for modern tech brands |
| Iconography style | Stroke weight, library preference, filled vs. line |
| Animation style | Easing curves, durations, entrance patterns |

**Reference brands:** When specifying visual direction, name 2–3 real brands that share the aesthetic (e.g., "Linear, Vercel, Vapi" for cyber-premium dark mode).

### 5. Social Media Design Direction

| Element | What to define |
|---------|----------------|
| Post styles | Single image, video, carousel, meme — each with rules |
| Carousel structure | Slide count, cover rules, progress indicators, final slide CTA |
| Ad styles | Headline length, visual rules, body copy length, button style |
| Thumbnail styles | Per-platform specs (YouTube, LinkedIn, Instagram) |
| Layout preferences | Asymmetric vs. symmetric, safe zones, text alignment |
| Visual hierarchy | Primary, secondary, tertiary elements |
| Text placement rules | Max words per line, safe margins, alignment |
| CTA placement | Bottom-right, center-bottom, embedded in video |
| Branding placement | Logo size, opacity, position — subtle but present |

### 6. Content Pillars

Define 4–8 content categories. For each:
- **Angle:** How this pillar is approached
- **Formats:** Carousel, video, screenshot, case study, etc.
- **Key message:** The one-line takeaway

Example pillars for an AI automation company:
- AI automation
- Ecommerce growth
- Lead generation
- AI agents
- Chatbot systems
- Case studies
- Conversion optimization
- Business scaling

### 7. Creative Psychology Rules

Document the mechanics of attention and conversion:

| Element | What to capture |
|---------|-----------------|
| Attention-grabbing | Pattern interrupts, specificity, visual contrast |
| Hook strategies | 6+ hook formulas with examples (confession, number, contrarian, question, roast, before/after) |
| Curiosity patterns | Open loops, withheld information, sequences, counterintuitive claims |
| Conversion messaging | One job per message, cost of inaction, ambiguity removal, proof stacking |
| Authority-building | Show the work, name specific tools, share failures, technical depth |
| Trust-building | Specificity, vulnerability, transparency, social proof density, risk reversal |

### 8. Example Brand Assets

Include concrete examples:
- 3–5 example captions (different platforms, different intents)
- 5–10 example hooks (one per hook strategy)
- 2–3 example ad copies (different platforms)
- 2–3 example post structures (roast, proof, education)
- 1 example carousel flow (slide-by-slide breakdown)
- 4–6 example CTA styles (direct, curiosity, exclusivity, risk reversal)

### 9. Video Script Writing (Short-Form Reels)

### The Funnel Structure

For short-form video (Reels, TikTok, Shorts), structure content across three funnel stages:

| Stage | Purpose | Tone | Length |
|-------|---------|------|--------|
| **Top of Funnel (TOF)** | Provoke awareness, create tension | Pain-first, observational, financially uncomfortable | 30–45s |
| **Middle of Funnel (MOF)** | Build trust, show legitimacy | Behind-the-scenes, failures, iteration | 45–60s |
| **Bottom of Funnel (BOF)** | Filter and convert | Premium, exclusive, qualification-first | 45–60s |

### The "Expensive Hook" Rule

**Never write educated hooks. Write expensive hooks.**

| Weak (Educated) | Strong (Expensive) |
|-----------------|-------------------|
| "Businesses lose money from slow response times" | "Most clinics lose leads while the receptionist is eating lunch" |
| "AI can improve your customer service" | "Your missed calls are probably your highest-intent leads" |
| "Follow up faster to win more deals" | "The first business to respond usually gets paid" |
| "Automation saves time and money" | "Hiring another receptionist is more expensive than fixing response time" |

**Expensive hooks are:**
- Specific, not generic
- Financially intelligent, not feature-focused
- Emotionally uncomfortable, not comfortable
- Observations, not claims
- Operator-level, not marketer-level

### Script Rhythm Rules

1. **One idea per sentence.** No compound explanations.
2. **Pause between beats.** Reels are retention warfare. Silence creates tension.
3. **Never sound memorized.** Write spoken, not written. Read aloud — if it sounds like a blog post, rewrite it.
4. **Cut defensive language.** Never say "Not a pitch," "Just my opinion," or "Follow for more." These destroy authority.
5. **End with observation, not request.** "Speed compounds" beats "Follow for more tips."

### What You're Actually Selling

**Never sell AI. Sell the business outcome AI enables:**
- Response speed
- Operational reliability
- Captured intent
- Reduced leakage
- Revenue continuity

AI is the mechanism. The more your script talks about money flow, operational friction, missed intent, and speed economics, the more dangerous your positioning becomes.

### Visual Direction for Reels

| Use | Avoid |
|-----|-------|
| Dashboards, call logs, CRM pipelines | Generic B-roll, stock footage of handshakes |
| Terminal screens, error logs, testing environments | Overedited cinematic fluff |
| Timestamps, metrics, dark UI | Talking head only, pointing at text |
| Actual workflow views, deployment clips | Random captions, meme formats |
| Laptop reflections, code, audio waveforms | Bright backgrounds, 3D mascots |

Your strongest asset is "builder energy." Show the work, not the polish.

### Remotion Video Production

For programmatic video generation (Reels, TikTok, Shorts), use Remotion to produce editor-ready compositions with stock footage, TTS audio, and programmatic screen mockups.

**Project Setup:**
```bash
npx create-video@latest project-name --blank --y
cd project-name
npm i
```

**Key Components to Build:**
- `VideoBackground` — darkened/blurred stock footage with fade transitions
- `TextOverlay` — animated text (fade, slideUp) with shadows
- `ScreenMockup` — programmatic UI: call logs, CRM pipelines, calculators, error logs, code editors
- `ProgressBar` — colored progress indicator per reel
- `AudioSegment` — audio with fade in/out

**Asset Pipeline:**
1. Generate TTS audio using edge-tts (free) or ElevenLabs API
2. Source stock footage from Pexels API (free, 200 req/hr) — requires API key
3. Download clips locally: `wget` or `curl` the CDN URLs
4. Place all assets in `public/` folder for Remotion staticFile access
5. Calculate frame durations from audio lengths: `frames = duration_seconds * fps`

**Composition Specs:**
- Resolution: 1080x1920 (vertical/reel format)
- FPS: 30
- Duration: calculated from audio segments
- Codec: h264 for MP4 output

**Stock Footage Categories to Source:**
- Phone ringing, office calls, missed calls
- Business handshake, frustrated customer
- Code screen, error messages, test fails
- Calculator, spreadsheet, CRM dashboard
- Dark office, laptop screen, typing

**Pitfall — Asset Path Resolution:**
Remotion's `staticFile()` resolves from the `public/` directory. If assets are in `src/assets/`, copy them to `public/` before rendering. The `staticFile("filename.mp4")` path must NOT include `public/` or `assets/` prefixes.

**Pitfall — Render Performance:**
Headless Chrome rendering on EC2 instances is slow (~40+ min per reel). For production:
- Render locally on a machine with GPU
- Or use Remotion Lambda for cloud rendering
- Or export the composition as a structured timeline for editor handoff

**Editor Handoff Format:**
When full Remotion render isn't feasible, provide:
1. Shot list with exact timestamps
2. Stock footage URLs/categories needed per segment
3. Screen mockup specifications (data to display)
4. Text overlay copy with timing
5. Audio files (TTS generated)

### Example: TOF Script — "The Lunch Break Leak"

```
[0:00-0:03] HOOK
"Most clinics lose leads while the receptionist is eating lunch."

[0:03-0:10] PROBLEM
"Think about it. Your highest-intent leads? They don't call at 10 AM on a Tuesday. They call at 1:15 PM. During lunch. After hours. On Sunday. And they don't leave voicemails. They call the next clinic on the list."

[0:10-0:17] OBSERVATION
"The first business to respond usually gets paid. Not the best. Not the cheapest. The fastest."

[0:17-0:28] TENSION
"They already decided to call you. That's the hard part. And most businesses still lose them. Response time is a revenue system. Most don't realize this until revenue drops."

[0:28-0:35] CLOSE
"Speed compounds."
```

**Visual Direction:**
- Screen recording of call log showing missed calls at 1:15 PM, 2:30 PM, Sunday
- CRM pipeline with deals marked "lost — no response"
- Timestamp overlays, dark UI, minimal text

### Example: MOF Script — "What Broke"

```
[0:00-0:03] HOOK
"I spent 3 weeks tuning an AI voice agent. Here's what broke."

[0:03-0:09] FAILURE 1
"First problem? It interrupted callers. Sounded efficient. Actually sounded rude."

[0:09-0:15] FAILURE 2
"Second problem? It failed with local accents. Testing looked perfect. Real calls were brutal."

[0:15-0:21] FAILURE 3
"Third problem? It answered price objections wrong. That mistake alone can kill deals."

[0:21-0:30] CLOSE
"Most demos break in real environments. This is the part most agencies never show."
```

**Visual Direction:**
- Terminal screen showing failed test logs
- Audio waveform of a call with interruption
- Screen recording of AI responding incorrectly to price question
- Dashboard with error rates spiking, dark IDE, real code

### Example: BOF Script — "Too Early"

```
[0:00-0:03] HOOK
"Most businesses are too early for this."

[0:03-0:10] FILTER
"This only works if you have consistent inbound demand. 20+ calls per day. A real sales process. Something to actually automate. If you're still figuring out where your leads come from, fix that first. This won't save you."

[0:10-0:20] ROI FRAME
"If you're getting 20+ calls and missing even 4 of them, do the math. What's your average deal worth? Multiply by 4. Multiply by 22 working days. That's your monthly leak. And that's just the calls you know about."

[0:20-0:32] POSITIONING
"This replaces response bottlenecks. Not marketing. Not sales skills. Just the gap between 'someone wants to talk' and 'someone actually talks to them.' That's it. That's all it does. But for the right business, that's everything."

[0:32-0:40] CLOSE
"If that gap is costing you money, DM me. I'll calculate the leak for your business."
```

**Visual Direction:**
- Spreadsheet with ROI math
- Call volume chart showing 20+ daily
- Pipeline view with bottleneck highlighted
- Dark theme, operational, no fluff

### Sequence Matters

Your first 10–15 posts should build:
1. Competence
2. Positioning
3. Perceived intelligence
4. Technical legitimacy

Then personality comes after. Authority first. Story later.

## 10. AI Generation Instructions

Clear instructions for AI systems producing content:

| Element | What to specify |
|---------|-----------------|
| Visual look | Default mood, background, text, accents, imagery style |
| Aesthetic reference | Genre, reference brands, mood, texture, motion |
| Emotional response | What each stage of engagement should feel like |
| Forbidden styles | What should NEVER be generated (stock photos, bright backgrounds, 3D mascots, etc.) |
| On-brand checklist | A pass/fail checklist for every asset |
| AI image prompts | 3–5 starter prompts for image generation tools |

## Design Principles

- **Prioritize visual consistency** over creative variety. One strong system beats ten weak variations.
- **Quantify everything.** "$43K recovered" beats "improved revenue."
- **Avoid generic branding language.** No "revolutionary," "cutting-edge," "world-class."
- **Write for the executor.** The reader is likely an AI system, a junior designer, or a social media manager who needs to produce assets without asking questions.
- **Include the negative space.** What NOT to do is as important as what TO do.
- **Reference real brands.** "Like Linear, not like Wix" is more useful than "modern and clean."

## Pitfalls

| Problem | Cause | Solution |
|---------|-------|----------|
| Document is too abstract | Wrote rules without examples | Every rule needs a good example and a bad example |
| Colors don't work together | Chose colors in isolation | Test contrast ratios, check against reference brands |
| Voice feels generic | Used corporate speak | Read it aloud — if it sounds like a press release, rewrite it |
| Visual section is ignored | Too much theory, not enough spec | Include exact HEX codes, font sizes, spacing values |
| AI generates off-brand content | Missing forbidden styles list | Explicitly list what should NEVER be generated |
| Document becomes outdated | No versioning or date | Include "Last updated" and commit to version control |
| Too long to be useful | Included everything | Prioritize — what does the executor need in the next 24 hours? |

## Workflow

1. **Gather inputs** — Ask the user for: brand name, mission, target audience, existing visual assets, competitor references, and what the document will be used for.
2. **Define the positioning** — Write Section 1 first. Everything else flows from this.
3. **Build the voice** — Write Section 3. The word lists (use/avoid) are the most reused part.
4. **Design the visual system** — Write Section 4. Start with colors and typography; everything else follows.
5. **Add tactical guidance** — Write Sections 5–7. These are the "how to execute" sections.
6. **Create examples** — Write Section 8. Examples validate that the system actually works.
7. **Add AI instructions** — Write Section 9. This makes the document future-proof.
8. **Review against checklist:**
   - [ ] Can a stranger produce on-brand content from this alone?
   - [ ] Are there specific numbers, not vague claims?
   - [ ] Is the visual system spec'd with exact values?
   - [ ] Are there examples for every abstract rule?
   - [ ] Is there a "what NOT to do" section?
   - [ ] Would this make a competitor nervous?
