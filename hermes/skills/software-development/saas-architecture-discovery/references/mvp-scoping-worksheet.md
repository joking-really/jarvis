# MVP Scoping Worksheet

Use this worksheet to force MVP prioritization when users describe full vision.

## The Rule

**MVP = 2-3 core features that differentiate + table stakes.**

Everything else is post-MVP. Be explicit. Get user agreement.

## Worksheet

### Step 1: List All Requested Features

Ask user: "What features do you want?" Let them list everything.

Example response:
- AI voice calling
- WhatsApp messaging
- Instagram automation
- Facebook Messenger
- Email
- Live dashboard
- Human handoff
- AI agent builder
- Visual flow builder
- Website auto-crawl
- Analytics
- White-label
- API
- Mobile app

### Step 2: Identify The Differentiator

Ask: "What feature would make a customer choose YOU over competitors?"

Example: "AI voice calling combined with WhatsApp and human handoff. No one does all three."

### Step 3: Identify Table Stakes

Ask: "What features do customers EXPECT any CRM to have?"

Example: "Contact management, conversation history, basic dashboard."

### Step 4: Propose MVP Scope

Present:
```
For MVP, I recommend:

IN SCOPE (launch in 12 weeks):
1. [Differentiator #1]
2. [Differentiator #2]
3. [Table stake #1]
4. [Table stake #2]

OUT OF SCOPE (post-MVP):
- [Feature]: Reason why deferred
- [Feature]: Reason why deferred
- [Feature]: Reason why deferred
```

### Step 5: Get Explicit Agreement

Ask: "Do you agree with this MVP scope? Any must-have I'm missing?"

### Step 6: Document Roadmap

```
MVP (Week 1-12): [features]
Month 3-4: [features]
Month 5-6: [features]
Month 7-12: [features]
```

## Example from Session

**User's full list:**
AI voice, WhatsApp, Instagram, Facebook, email, live dashboard, human handoff, AI agent builder, visual flow builder, website auto-crawl, analytics, white-label, API, mobile app

**Differentiator:** AI voice + multi-channel + human handoff

**Table stakes:** Contact management, conversation timeline, basic dashboard

**MVP proposed:**
- Voice (inbound + outbound)
- WhatsApp
- Email
- Human handoff
- Live monitoring
- Contact management
- AI agent configuration (3 templates)
- Basic dashboard

**Deferred:**
- Instagram/Facebook: API restrictions, lower priority than WhatsApp in Pakistan
- Visual flow builder: Complex, 4-6 weeks dev time
- Website auto-crawl: Requires NLP, anti-bot handling
- Advanced analytics: Operational metrics sufficient for MVP
- Mobile app: PWA sufficient initially
- Full white-label: Logo + color only for Pro

**User agreement:** "Okay, then for the MVP, we should take voice, WhatsApp, plus email as the most important things."

## Pitfalls to Avoid

| Pitfall | Why It Happens | Solution |
|---------|---------------|----------|
| User insists on everything in MVP | Enthusiasm, fear of missing features | Show timeline impact: "All features = 6 months. MVP = 12 weeks." |
| User adds features after agreeing to MVP scope | Re-evaluating, new ideas | Remind of locked scope: "We agreed MVP is X. This is a great post-MVP feature." |
| Competitor has feature user wants | FOMO | "They have 50 engineers and 5 years. We have 12 weeks. Let's nail the core first." |
| User underestimates complexity | Non-technical background | Break down into dev weeks: "This feature = 3 weeks of one developer's time." |
| "But it's just a small feature" | Scope creep justification | "Small features add up. 5 'small' features = 10 weeks." |

## Decision Log Entry

After MVP scope is agreed, add to decision log:

```
## Decision: MVP Feature Scope

**Date:** YYYY-MM-DD
**Status:** ACCEPTED
**Context:** 12-week launch timeline, limited dev resources

**In Scope:** [list]
**Out of Scope:** [list with reasons]
**Post-MVP Roadmap:** [phased timeline]

**User Agreement:** [quote or summary]
```
